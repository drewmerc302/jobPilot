### The jobspy/numpy problem

`python-jobspy >=1.1.75` pins `NUMPY==1.26.3` which has no cp313 binary wheel. Briefcase macOS enforces `--only-binary :all:`. So pip can't install jobspy >=1.1.75.

**Workaround** (codified in `scripts/build_dmg.sh`):
1. Let Briefcase install old jobspy 1.1.13 (resolves because it doesn't pin numpy strictly)
2. After build, swap `jobspy` + `tls_client` + `markdownify` + `regex` from the dev venv (which compiled numpy from source)
3. Then package

**Dev venv bootstrap** (one-time):
```bash
briefcase dev          # creates .briefcase/jobpilot/dev.cpython-313-darwin/
# Then in that venv:
.briefcase/jobpilot/dev.cpython-313-darwin/bin/pip install python-jobspy
```

### Why old jobspy (1.1.13) didn't work
- Missing `hours_old` parameter in `scrape_jobs()` — our `jobspy_scraper.py` uses it
- Call threw `TypeError`, caught by broad `except Exception` in `_scrape_one()`, silently returned 0 jobs
- This was the root cause of "50 jobs scraped but 0 in DB" in the first test

## Known issues NOT yet fixed

1. **Version number**: Still `0.0.1` in `pyproject.toml` — bump when ready to ship
2. **No Greenhouse warnings in wizard**: Wizard's `_build_scrapers` doesn't surface skipped companies to the UI (matches.py does via `run_status[run_id]["warnings"]`)
3. **Location filter geocoding**: `latitude`/`longitude` are null in search params — geocoding from zip code happens in pipeline via `geocode(search_params)` but may not persist back to JSON
4. **`remote` vs `is_remote`**: DB column is `remote`, location_filter checks `is_remote` — currently harmless because lat/long being null skips location filter entirely, but will break when geocoding works
5. **Cost of Greenhouse LLM filter**: With 14 anchor companies, Greenhouse returns ~1000-2000 jobs. LLM-filtering all of them costs ~$1-2 per run. May want to add a pre-filter by department/title before LLM step to reduce cost
6. **pyproject.toml still says `>=1.1`** for python-jobspy — the build script handles the workaround, but the requirement doesn't reflect what actually runs
